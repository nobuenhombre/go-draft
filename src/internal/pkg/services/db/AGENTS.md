# db — Генерация bash-скриптов базы данных

## Назначение

Пакет `db` генерирует bash-скрипты для управления базой данных PostgreSQL: миграции, бэкапы, создание, кодогенерация через xo.

## Файлы

| Файл | Назначение |
|------|------------|
| `db.go` | `Service` (интерфейс), `Provider` (реализация), `New()`, `CreateDb()`, поиск шаблонов |
| `provider.go` | Wire-провайдер: `ProviderSet` + `ProvideDbService` |

## Типы

- **`Service`** — интерфейс с методом `CreateDb(dbName string) error`
- **`Provider`** — реализация (stateless)
- **`TemplateData`** — `DbName` + `ModulePath` для выполнения шаблонов

## Поведение

`CreateDb("fastmail_gpt_reply")`:
1. Определяет корень проекта (где `go.mod`)
2. Читает module path из `go.mod`
3. Ищет `templates/db/` в стандартных путях
4. Обходит все `.tpl` файлы рекурсивно
5. Для каждого:
   - Выполняет `text/template` с `{DbName, ModulePath}`
   - Если файл уже существует — пропускает (shared scripts не перезаписываются)
   - `.sh` файлы создаются с правами `0755`
   - Остальные — `0644`

## Генерируемые файлы

Для первой БД — 12 файлов:

```
src/scripts/xo/
├── xo.sh              # общие утилиты (7, только при первом запуске)
├── yaml.sh
├── postgresql.sh
├── backup.sh
├── restore.sh
├── create.sh
├── lint.sh
└── {DbName}/
    ├── Makefile
    ├── xo.yaml
    ├── migrate-up.sh
    ├── migrate-down.sh
    └── migrate-new.sh
```

Для второй и последующих БД — только 5 файлов в `{DbName}/` (общие скрипты уже существуют).

## Поиск шаблонов

Аналогично `dirs` и `app` — через `locator.FindTemplateDir("db")`.

## Переменные шаблонов

| Переменная | Описание | Источник |
|------------|----------|----------|
| `{{.DbName}}` | Имя базы данных | Флаг `-dbname` |
| `{{.ModulePath}}` | Go module path | Из `go.mod` проекта |

## Правила

- Шаблоны — чистые bash-скрипты и конфиги (не Go)
- Если файл уже существует на диске — не перезаписывается
- `.sh` файлы создаются исполняемыми
- Ошибки оборачивать через `ge.Pin(err)`
