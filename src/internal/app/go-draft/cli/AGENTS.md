# cli — Парсинг CLI-флагов

## Назначение

Пакет `cli` отвечает за разбор аргументов командной строки приложения. Использует библиотеку `suikat/clivar` для автоматического маппинга CLI-флагов в структуру через теги.

## Структура

- **`cli.go`** — основной код
- **`provider.go`** — Wire-провайдер (`ProviderSet` + `ProvideCLI`)

## Типы

- **`Service`** — пустой интерфейс-маркер, через который `Config` передаётся по слоям через Wire DI
- **`Config`** — структура с CLI-параметрами, определёнными через теги `cli:"..."`

## CLI-флаги

| Флаг | Тип | По умолчанию | Описание |
|------|-----|-------------|----------|
| `-make` | string | `dirs` | Команда для выполнения (`dirs`) |
| `-dirs` | string | `classic` | Имя шаблона структуры директорий |
| `-vars` | string | `""` | Переменные для подстановки в формате `key1:val1,key2:val2` |

## Константы команд

- `MakeDirs = "dirs"` — создание структуры директорий по шаблону

## Wire-интеграция

Wire-провайдер `ProvideCLI()` в `provider.go`:
1. Вызывает `cli.New()` — парсит аргументы командной строки
2. Возвращает `(Service, func(), error)`

Потребители получают `cli.Service` и делают type assertion к `*cli.Config`.

## Правила изменения

- Теги `cli:"..."` определяют имя флага, описание, тип и значение по умолчанию
- При добавлении нового флага: добавить поле в `Config` с тегом `cli`
- Интерфейс `Service` намеренно пустой — не добавлять в него методы, он служит маркером для DI
- Ошибки оборачивать через `ge.Pin(err)`
